Learning roadmap

Learn it in this order

The clean mental model is: app config → navigation → launcher/search/sections → list data → cards → record screens → actions → simulator validation.

1. Start with the mobile component hierarchy

The demo introduces the Now Mobile configuration as a tree of objects: app config, navigation bar, tabs, launcher screens, screens, screen segments, item streams, data items, item configurations, and functions.

  • Treat the mobile app config as the root.
  • Navigation tabs decide what the user sees at the bottom of the app.
  • Launcher screens are landing pages; record/list screens are task or object-specific pages.
  • Functions are reusable actions or navigation targets attached to screens, cards, and buttons.

2. Understand the Mobile App Builder shell

The builder shows the mobile app tree on the left, the editable selected record in the center, and a mobile interface example on the right. The iPhone simulator gives the real runtime view.

  • Left tree = configuration structure.
  • Center form = selected artifact's properties, data, conditions, and relationships.
  • Right preview = conceptual rendering for that artifact type.
  • Simulator = actual experience for the user.

3. Configure global navigation

The Now Mobile navigation contains tabs such as Home, Support, Notifications, and Settings. The demo also opens the Notifications tab to show label, active flag, and icon configuration.

  • Navigation bar tabs are ordered: Home, Support, Notifications, Settings.
  • Each tab references a launcher screen or a special tab behavior.
  • The notification tab has its own label, active flag, and icon.
  • Changing navigation structure changes the bottom mobile tab experience.

4. Wire the launcher screen

A launcher screen is built from a header, global search, and UI sections. In the demo, the Home and Support tabs are launcher-style pages with cards, sections, and search experiences.

  • Header gives the user context.
  • Global search sits near the top and uses a search configuration.
  • UI sections arrange records/cards such as Quick links, My items, Recommended for you, Popular topics, and Browse all topics.
  • Each section can point to a destination screen or open a function.

5. Learn screen vs screen segment

A screen is the full page or list; a screen segment is a reusable area inside a screen, such as details, updates, fields, or activity.

  • List screens show many records.
  • Record screens show one record.
  • Segments divide a screen into tabs or regions like Details and Updates.
  • Segments can host fields, activity streams, and function instances.

6. Build list experiences from stream → data item → list item config

The My Tasks example shows an item stream named Approval To-do, a data item using Approval [sysapproval_approver], and separate list item configs for REQ and RITM approval rows.

  • Item stream selects a table and the record feed.
  • Data item defines the data source.
  • List item config decides row layout and behavior.
  • Conditions decide which list item config applies when multiple configurations exist.

7. Configure approval details and actions

The approval details screen includes fields, a Details screen segment, field function instances, and footer function instances. The runtime shows buttons such as Reject, Approve & Comment, and Approve.

  • The record screen uses the same source table as the list item.
  • Screen fields expose key approval/request fields.
  • Footer function instances become primary/secondary action buttons.
  • Function records can navigate, submit actions, or collect comments.

8. Customize cards in Mobile Card Builder

The Catalog Approval Template is edited in Mobile Card Builder. Elements such as approval text, number, price label, price, requested-for avatar, and requested-for text are arranged into containers.

  • Card templates are flexible layouts for mobile list rows and record cards.
  • UI rules can switch visibility, labels, or values by condition.
  • Card element IDs help map builder elements to data and styling.
  • Preview the template against the simulator to avoid layout mismatch.

9. Connect search and topic sections

Search configuration uses a search engine and multiple sources. The demo shows sources like Services, Articles, and People, then configures launcher sections such as Recommended for you, Popular topics, and Browse all topics.

  • Search application configuration controls the search engine and result limits.
  • Application search sources determine which content types appear.
  • Support launcher sections can be horizontal, free-scroll, capped by item count, and hidden if empty.
  • Each topic/card can open a destination screen or function.

10. Validate runtime behavior in the mobile simulator

The demo frequently switches between configuration and the iPhone simulator. This is important because some preview panes are abstract while the simulator shows the actual mobile UX.

  • Refresh/reopen the app after configuration changes.
  • Test the bottom navigation tabs.
  • Open list rows, record screens, and topic pages.
  • Check buttons, detail sections, search results, and catalog item pages.

11. Know where Service Portal still participates

Near the end, the demo opens the Service Portal SC Catalog Item widget. The widget contains native-mobile-aware HTML and client script behavior such as attachments, order button logic, breadcrumbs, and native mobile options.

  • Mobile catalog rendering can still involve Service Portal widgets.
  • The SC Catalog Item widget checks native mobile options.
  • HTML and client script influence the mobile catalog item experience.
  • Do not assume Mobile App Builder is the only layer involved.

Important distinction

A screen is a full mobile page. A screen segment is a region inside that page. A list item config controls how rows render. A function controls what happens when the user taps something.

ArtifactMeaning in the demo
Mobile app configRoot configuration for Now Mobile. It groups navigation, theme, app behavior, and child configuration records.
Navigation barContainer for bottom tabs. In the demo, the Now Mobile Nav contains Home, Support, Notifications, and Settings.
Navigation tabA tab shown at the bottom of the mobile app. It usually points to a launcher screen or special tab function.
Launcher screenLanding screen with header, search, and UI sections. Example: Home and Support.
Record sectionA section on a launcher screen showing cards or results. Example: Recommended for you, Popular topics, Browse all topics.
Search configDefines search behavior, source list, empty states, result limit, and suggestion limit.
List screenA screen that displays a list of records; usually backed by an item stream.
Item streamDefines the record stream/table and possibly interleaving/order behavior.
Data itemThe data source for a stream or screen.
List item configDefines how each row is rendered and when the configuration applies.
Card / card templateVisual template for rows/cards, edited in Mobile Card Builder.
Record screenA detail page for one record such as a RITM, request, topic, or approval.
Screen segmentA subsection inside a screen, such as Details, Updates, Activity, or field areas.
FunctionAn action or navigation behavior. Examples: open catalog item, open topic page, jump to RITM, approve, reject, approve with comment.

Configuration checklist

Build checklist

  • Create or choose the mobile app config.
  • Verify navigation bar tabs and their order.
  • Attach launcher screens to tabs such as Home and Support.
  • Configure global search and search sources.
  • Add record sections like Recommended for you and Popular topics.
  • Create list screens with item streams and data items.
  • Add list item configs and card templates for row display.
  • Create record screens and screen segments for details.
  • Add function instances for open, approve, reject, comment, and navigation.
  • Validate every path in the mobile simulator.

Debugging checklist

When something does not show

  • Check the selected application scope and original scope warning.
  • Confirm the record is Active.
  • Check visibility conditions and filter conditions.
  • Check whether section is hidden when empty.
  • Check item stream table and data item source.
  • Check list item config conditions when multiple configs exist.
  • Open Mobile Card Builder if the card content is wrong.
  • Check function context and link URL if tapping does nothing.
  • Check Service Portal widget code for catalog item rendering issues.
  • Refresh/reopen simulator after changes.
