Variables System

glide-servicecatalog Runtime ModelsPersistenceViewsRhino Bridge

1. Architecture Overview

The variable system is a three-layer hierarchical document model that handles both single-value fields and multi-row table-like variable sets.

┌─────────────────────────────────────────────────────┐
│  Presentation Layer                                  │
│  TableVariableViewImpl / TableVariableRowViewImpl    │
│  TableVariableCellView / TableVariableJsonModel      │
└───────────────────┬─────────────────────────────────┘
                    │ reads
┌───────────────────▼─────────────────────────────────┐
│  Runtime Layer                                       │
│  TableVariableNode / TableVariableRowNode            │
│  TableVariableCellNode / VariableNodeScriptingModel  │
└───────────────────┬─────────────────────────────────┘
                    │ reads/writes
┌───────────────────▼─────────────────────────────────┐
│  Persistence Layer                                   │
│  VariableNodeDatabaseModel                           │
│  Table: sc_multi_row_question_answer                 │
└─────────────────────────────────────────────────────┘

2. Variable Set Types

VariableSetType Enum

Type Class Behavior
ONE_TO_ONE SingleRowVariableSet Single row of fields — standard form fields
ONE_TO_MANY TableVariableSet Multi-row table — each row stores independent values

Factory method: VariableSetType.get(type) returns the appropriate AbstractVariableSet implementation.

3. Persistence Layer

VariableNodeDatabaseModel

Purpose: In-memory document model for all multi-row variable data. Bridges between the application layer and sc_multi_row_question_answer DB table.

Table: sc_multi_row_question_answer

Column Purpose
node_owner_id Source record sys_id (sc_req_item, sc_task)
node_owner_table Source table name
node_parent_id Parent node sys_id (for hierarchy)
node_question_id Question definition sys_id
node_type "table", "tr" (row), "td" (cell)
node_variable_set_id Variable set definition sys_id
node_value Serialized value
node_index System.nanoTime() — row ordering

Loading Process

VariableNodeDatabaseModel.getDocumentRoot(variableSetId, ownerId, ownerTable)
  1. Query sc_multi_row_question_answer filtered by variable set + owner
  2. Build in-memory tree: root → row nodes (tr) → cell nodes (td)
  3. Map question IDs per variable set via HashMap for rapid lookup
  4. Create IVariableNodeContext with nodeMap + query cache

Saving Process

VariableNodeDatabaseModel.save()
  1. Iterate nodeMap for modified nodes (marked via markDirty())
  2. New nodes (temp UUID prefix):
     → INSERT to sc_multi_row_question_answer
  3. Existing nodes:
     → UPDATE value + display_value fields
  4. Deleted nodes:
     → DELETE by sys_id
  5. getQuestionReportingMapping()
     → establish references to sc_item_option / question_answer source records

Query System (Visitor Pattern)

IQuery
  ├── EqualsQuery(field, value)     → single equality condition
  ├── AndQuery(List<IQuery>)        → all conditions must match
  └── OrQuery(List<IQuery>)         → any condition matches

Query cache: fQueryResultCache keyed by cacheKey() string. Cleared by markDirty() on any modification.

Database translation: QueryConditionBuilder extends BaseQueryVisitor<IQueryCondition> - EqualsQuery → addCondition(field, "=", value) - AndQuery → chained addCondition() - OrQuery → addOrCondition()

4. Runtime Layer

Node Hierarchy

ITableVariableNode (root — "table")
    └── ITableVariableRowNode (row — "tr")
              └── ITableVariableCellNode (cell — "td")

TableVariableNode

Purpose: Root node for a variable set instance.

Key Fields: | Field | Purpose | |-------|---------| | fAttributes | HashMap — variable set metadata (max_rows, layout, type, roles) | | fQuestionSysIds | LinkedHashMap — ordered question sys_id → definition ID |

Key Methods: | Method | Purpose | |--------|---------| | addRow() | Create new row; index = System.nanoTime() | | deleteRow(rowId) | Soft-delete row node | | getMaxAllowedRows() | min(system property glide.sc.multirow_set.rows.size, max_rows attribute) — returns -1 if unbounded | | getRows() | Return ordered list of row nodes |

TableVariableRowNode

Purpose: Represents one row in a multi-row variable set.

Key Methods: | Method | Purpose | |--------|---------| | getCell(questionName) | Return cell node by question name | | deleteRow() | Mark row as deleted (soft delete) | | getIndex() | Return nanotime ordering index |

TableVariableCellNode

Purpose: Leaf node representing one field value in a row.

Key Methods: | Method | Purpose | |--------|---------| | getCellValue() | Return raw persisted value from node | | getCellDisplayValue() | Compute display value via Question (handles derived fields, reference labels) | | isHideOnGrid() | Delegates to Question.isHideOnGrid() |

VariableNodeScriptingModel

Extends: GlideElementScriptingModel
Implements: IGlideElementWiringCallback

Purpose: Bridges the runtime node model to the Rhino JavaScript environment.

jsFunction_ methods (callable from catalog scripts):

variableNode.getRows()
variableNode.addRow()
variableNode.deleteRow(rowId)
variableNode.getCellValue(rowId, questionName)
variableNode.setCellValue(rowId, questionName, value)

commit() hook: Fires when a script writes to the variable (via gr.insert/update/delete). Routes to save() for non-DELETE operations.

5. View Layer

View Construction Pipeline

VariableSetContainer
    ↓ resolveModel()
ITableVariableNode (loaded from VariableNodeDatabaseModel)
    ↓ getTableVariableView()
TableVariableViewImpl
    ├── TableVariableRowViewImpl (per row)
    │       └── TableVariableCellView (per cell)
    └── ... (all rows)

TableVariableViewImpl

Constructor:

TableVariableViewImpl(VariableSetContainer container, String sourceTable, String sourceId)
  1. Resolve ITableVariableNode via IVariableModelInstanceProvider
  2. Iterate rows → build TableVariableRowViewImpl for each

TableVariableRowViewImpl

Constructor:

TableVariableRowViewImpl(List<String> questionIds, ITableVariableRowNode rowNode)
  1. Map each questionId → TableVariableCellView via row.getCell(question.getName())
  2. Store question metadata: name, label, order

TableVariableCellView

Purpose: Single-cell view wrapping ITableVariableCellNode + Question.

Method Behavior
getValue() Returns raw persisted value
getDisplayValue() Evaluates via Question; caches on first access

Special type handling: | Type | Behavior | |------|----------| | DURATION | Converts via GlideDuration().setValue(value) | | DATE / DATE_TIME | Uses display value only (no parsing) | | Others | Raw persisted value |

TableVariableJsonModel

Purpose: Serializes all rows/cells to JSON for API/client consumption.

Output structure:

[
  {
    "questionId1": { "value": "...", "displayValue": "..." },
    "questionId2": { "value": "...", "displayValue": "..." }
  },
  ...
]

Type-specific JSON serialization: - DATE/DATE_TIME → displayValue for both value and displayValue fields - DURATION → convert via GlideDuration.getValue() - Others → raw persisted value

6. Variable Set Implementations

AbstractVariableSet

Base class for all variable set types.

Key Methods: | Method | Purpose | |--------|---------| | load() | Initialize and populate questions | | getOnChange() | Return on-change JS hook string | | getQuestionValue(q) | Read from fRowData JSONObject by question ID/name | | updateQuestion(q) | Apply row data to question with type-safe conversion | | getRefResolverRecord() | Create transient sc_cart_item for reference qualifier evaluation |

SingleRowVariableSet (ONE_TO_ONE)

  • Queries item_option_new for variable definitions
  • Wraps questions in VariableSetContainer
  • Questions rendered as standard form fields
  • No multi-row persistence overhead

TableVariableSet (ONE_TO_MANY)

Key Additional Fields: | Field | Purpose | |-------|---------| | fRowData | JSONObject of current row's values | | fFetchQuestionsFromModel | Whether to load questions from node model |

Key Additional Methods: | Method | Purpose | |--------|---------| | setRowData(jsonData) | Set current row context for question value resolution | | setSequence(varIds) | Override question display order | | setFetchQuestionsFromModel(bool) | Control question source |

On-change hook template:

if (typeof(variableOnChange) == 'function') variableOnChange('{questionName}')

Type conversions in updateQuestion(): | Type | Conversion | |------|-----------| | DATE / DATE_TIME | q.setDisplayValue(value) — no parsing | | DURATION | new GlideDuration().setValue(value) → q.setValue(duration.getValue()) | | Others | q.setValue(value) |

7. Scripting Infrastructure

Package: com.glide.catalog.scripting.rhino

IRhinoScriptingEnablement (Interface)

Minimal contract for Rhino property access on Java objects:

Class[] getPublicScriptableInterfaces()
Object jsGet(int index)     // obj[0]
Object jsGet(String name)   // obj.name
Object jsGet()              // used as default value
void jsSet(int index, Object value)
void jsSet(String name, Object value)
void jsSet(Object value)

GlideElementScriptingModel

Extends: GlideElementGlideObject
Implements: IRhinoScriptingEnablement, IScriptingHelp, IGlideElement, IJsonDataModel

Purpose: Core Rhino bridge — makes Java objects accessible from JavaScript with natural property/method syntax.

Method invocation flow:

JavaScript: obj.propertyName
    ↓
Rhino calls: get(String name, Scriptable start)
    ↓
Check: is property in fMappedNames?
    ↓
Call jsGet(String name)
    ↓
Find matching method by parameter count (reflection)
    ↓
Invoke via reflection
    ↓
Wrap result: NativeArray for arrays, else direct
    ↓
Return to Rhino

Annotations processed at construction: | Annotation | Purpose | |-----------|---------| | @DefaultJavaScriptGetter | Mark method as default Rhino getter | | @DefaultJavaScriptSetter | Mark method as default Rhino setter |

Special handling: - toString() — prevents NativeObject type-coercion errors - Array returns — wrapped in NativeArray for JavaScript iteration - Method logging — when ServiceCatalogToggles.isVariableScriptingDebuggingEnabled()

IGlideElementWiringCallback (Interface)

Minimal decoupling layer between scripting model and underlying field implementation:

void configureElementDescriptor(ElementDescriptor descriptor)
String stringValue()        // → getValue()
String scriptVariableName() // → getName()
boolean hasChanges()        // → changes()
void commit(String op)      // → notifyRecordOperation()
IVariable variable()        // → IVariable for compatibility

9. Question Types and Special Handling

Reference / Lookup Variables

  • LookupSelectQuestion requires reference context for resolving reference qualifiers
  • AbstractVariableSet.getRefResolverRecord() creates a transient sc_cart_item record
  • Stores initial question values to provide context for qualifier evaluation
  • LookupSelectQuestion.setReferenceContext() called during load()

Hidden Grid Columns

  • Questions with hideOnGrid=true excluded from table column rendering
  • Still participate in cell data storage and scripting access
  • ITableVariableCellNode.isHideOnGrid() → Question.isHideOnGrid()

List Collector

  • Returns DISPLAY_VALUE_LIST as a separate field in API responses
  • Serialized as comma-separated display values

Masked Variables

  • VariablePoolQuestionSetJS returns DECRYPTED_VALUE as a separate API field
  • Raw encrypted value stored in VALUE

10. Multi-Row Variable Set: Max Row Enforcement

TableVariableNode.getMaxAllowedRows()
  systemPropLimit = getProperty("glide.sc.multirow_set.rows.size")
  attrLimit = fAttributes.get("max_rows")
  return min(systemPropLimit, attrLimit)
  // returns -1 if no limit (unbounded)

Enforced at row addition time, not at persistence time. UI prevents adding rows beyond limit.
