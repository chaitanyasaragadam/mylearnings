How to use this guide

Read the modules in order if you are new to Virtual Agent. If you already know VA Designer, jump directly to the Request Catalog Item topic block and backend support rules sections.

Core idea: Service Catalog in Virtual Agent is not only a chat UI. It is a decision-driven render engine that decides whether each catalog item and variable can run in conversation, popup, or full-page window mode.

1. Platform overview

The session starts by positioning Virtual Agent as a conversational layer for Service Catalog. The main pieces shown are the conversational interface, Virtual Agent Designer, and supporting plugins. Live Agent Support is mentioned as outside this demo scope.

What to learn

  • Virtual Agent has two sides: the user-facing chat interface and the designer/admin build experience.
  • A topic is a conversation blueprint; a topic block is reusable logic that can be called from a topic.
  • Topic blocks are not independently discoverable, so they generally do not need user-facing NLU keywords.

What to observe in the demo

  • Glide Virtual Agent plugin
  • Service Management Virtual Agent Topic Blocks plugin
  • difference between a topic and a topic block
05:00–17:00

2. Build a simple topic in Virtual Agent Designer

The demo creates a custom topic using out-of-box components. It starts from an empty Start → End canvas, adds a text input, stores values in variables, sends bot responses, uses a boolean/decision branch, adds script variables, and tests the topic in the preview window.

What to learn

  • Designer uses drag-and-drop nodes from a left palette.
  • User Input nodes collect values and create variables.
  • Bot Response nodes display text, images, cards, links, HTML, tables, or scripts depending on need.
  • Decision nodes split the conversation into true/false or branch-based paths.
  • The Test modal lets you validate a topic before publishing.

What to observe in the demo

  • Node name vs variable name
  • Prompt field
  • Input format / NLU entity
  • Include topic discovery checkbox
  • Variables tab in preview
18:00–20:00

3. Understand VA render types for catalog items

The demo explains why catalog item questions do not always render the same way in chat. Virtual Agent can render eligible variables directly in the conversation, open a popup for semi-supported cases, or navigate to a window when the item is unsupported for chat rendering.

What to learn

  • Conversation render stays inside the VA chat window.
  • Popup render opens a seamless modal-style experience without sending the user to a separate browser tab.
  • Window render navigates the user to a full page when the item cannot be safely rendered inside the chat.

What to observe in the demo

  • Supported variables
  • non-scripted UI policies
  • unsupported variables
  • content item/order guide conditions

4. Test the topic from Employee Center / Service Portal

The user-facing side is demonstrated from the Employee Center portal. The assistant opens as Now Support, shows open issues, handles typed input, provides topic discovery, and falls back when it cannot understand a request.

What to learn

  • The same topic can be tested in the Designer preview and from the actual portal chat widget.
  • Topic discovery can display suggested topics or a browse-all option.
  • Poorly matched user input triggers a fallback prompt asking the user to rephrase.

What to observe in the demo

  • Now Support chat launcher
  • Show me everything button
  • open issue messages
  • fallback/no-match wording

5. Request Catalog Item topic block

The large out-of-box Request Catalog Item topic block is inspected. It takes input variables, validates the selected item, determines the right render path, asks supported questions, handles attachments, and proceeds toward preview/order/checkout behavior.

What to learn

  • The topic block encapsulates reusable catalog ordering logic.
  • Input variables include catalog item id and flags such as proceed, add attachment, order item, submit producer, and upload attachment.
  • Multiple decisions handle supported/unsupported question types and whether to preview, order, or route to another UI.

What to observe in the demo

  • Request Catalog Item topic block
  • Script action utility nodes
  • Decision nodes
  • Checkout Cart branch
  • Submit Producer vs Order Item paths

6. Catalog item configuration that affects VA behavior

The demo jumps into catalog item records such as Apple iPhone 13 Pro and Microsoft Access. It shows variables, variable sets, Show VA render type, reference questions, date questions, and how values are collected and confirmed in the chat.

What to learn

  • Catalog variables are not just form fields; they determine how VA must ask questions.
  • Reference variables can render as a searchable selector when supported.
  • Date variables can be rendered inside chat as selectable choices.
  • Variable sets attached to catalog items must also be assessed for VA support.

What to observe in the demo

  • Variables related list
  • Variable Sets related list
  • Show VA render type related link
  • Reference variable question
  • date question
23:00–46:00

7. Script actions and checkout internals

Several Script Action Utility nodes are opened. These scripts validate that the catalog item exists and is visible, serialize user answers into item JSON, execute dynamic value logic, reset validation flags, and perform cart checkout to create request records.

What to learn

  • Script action nodes can read VA inputs with vaInputs and write shared values using vaVars.
  • Validation scripts prevent invisible or invalid catalog items from continuing.
  • Post-response logic stores answers as item JSON and can trigger dynamic values for dependent variables.
  • Checkout logic uses a cart request, then retrieves request/item records and moves attachments as required.

What to observe in the demo

  • vaInputs.catalog_item_id
  • GlideappCatalogItem.get()
  • catItem.canView()
  • itemJSON
  • cartObj.checkoutCart(true)
  • sc_request and sc_req_item lookup
47:00–58:00

8. Backend support rules in code

The last part moves into code to explain how catalog items are converted into a conversation object and how VA support is computed. The code checks catalog class support, variable support, UI policies, client scripts, data lookups, dynamic values, reference qualifiers, and question limits.

What to learn

  • CatalogConversationHelperJS builds a conversation object from a catalog item.
  • Unsupported catalog classes include content item, wizard, standard change, and order guide style classes.
  • questionLimit constrains how many flat questions/containers can render conversationally.
  • ReferenceQuestion.hasVASupport rejects unsupported reference qualifier cases and high-choice-count scenarios.
  • Dynamic value code must respect ordering so lower-order variables are not modified by higher-order variables in unsupported ways.

What to observe in the demo

  • SC_PROP_CONVERSATIONAL_REQUEST_QUESTION_LIMIT
  • SC_PROP_VA_REF_QUESTION_CHOICES_LIMIT
  • hasVASupportedUIPolicies
  • hasVASupportedDataLookups
  • hasVASupportedDynamicValue
  • supportedQuestion(q)

Implementation checklist

Use this as a quick review checklist when designing Service Catalog experiences for Virtual Agent.

AreaWhat to verify
Topic setupCreate topic, add NLU/topic discovery only when the topic should be discoverable.
Input collectionUse Text, Boolean, Static Choice, Reference, Date/Time, File Picker, or other supported User Input nodes.
Variable handlingUse input variables for user-provided values and script variables for computed/internal values.
Bot responseUse Text/HTML/Card/Table/Script responses to guide the user.
BranchingUse Decision nodes to separate yes/no, supported/unsupported, preview/order, and attachment paths.
TestingUse Designer preview first, then portal chat to validate real user experience.
Catalog renderCheck Show VA render type to confirm Conversation, Popup, or Window behavior.
Support rulesReview unsupported client scripts, scripted UI policies, data lookups, dynamic values, and unsupported variable types.
CheckoutValidate the item, collect variables, create cart item, checkout cart, and map request/RITM records.
