What This Module Is

glide-servicecatalog is the core ServiceNow Service Catalog platform. It handles everything from catalog/category browsing through item ordering, cart management, request fulfillment, record production, variable management, and delivery planning. It is an enterprise-grade, multi-tenant Java module with ~551 Java source files.

Module Composition: Two Root Packages

Package Role
com.glideapp.servicecatalog Domain layer — catalog entities, business rules, cart/checkout, order guides, pricing
com.glide.catalog Framework layer — caching infrastructure, scripting engine, variables runtime, component registry, UI policies

Core Domain Model Hierarchy

CatalogRecord (abstract base — wraps GlideRecord)
    ├── Catalog                  ← Top-level catalog container
    ├── Category                 ← Hierarchical item organizer
    └── CatalogItemImpl (abstract)
            ├── CatalogItem      ← The primary offering (2664 lines)
            ├── CartItem         ← Item placed in shopping cart
            └── RequestItem      ← Line item in a submitted request

Unifying interface: ICatalogItem — shared contract across CatalogItem, CartItem, RequestItem: - getCatalogItem(), getPrice(), getQuantity(), getOrderGuideRule()

Constants contract: ICatalogConstants (777 lines) — all table names, field names, system property keys

End-to-End Order Flow

1. Browse
   Catalog → Category.getItems() → CatalogItem list

2. Item Selection
   CatalogItem.canOrder() → load variables (Questions)
   → evaluate UI policies → show form

3. Cart
   Cart.addToCart(itemId) → CartItem created
   → variables captured via Questions
   → OrderGuide sequences additional items (if applicable)

4. Checkout
   CartValve → ProducerValve pipeline
   → Template applied (if any)
   → Script chain: SaveScript → PreScript → DB insert → PostScript
   → sc_item_produced_record logged

5. Request Created
   Request + RequestItem records
   → recalculatePrice() + recalculateRecurringCost()
   → DeliveryPlan attached (BasicPlanLoader / AdvancedPlanLoader)
   → TaskToken graph computed for delivery scheduling

Key Design Patterns

Pattern Location Purpose
Valve / Chain of Responsibility ValveProcessor → Valves HTTP request processing pipeline
Singleton CatalogUtil, CatalogSearch Thread-safe global utilities
Strategy PriceGenerator, IAnswerGenerator, VariableSetType Pluggable algorithms
Factory CatalogObjectCreator, CatalogUtil Object creation abstraction
Observer CatalogUpdateEventGenerator + DB listeners Event-driven cache invalidation
Composite CompositeProducerValve Multi-record production as one unit
Decorator CatalogItemImpl wrapping GlideRecord Add behavior without modifying base
Template Method CatalogTemplate, RenderCatalog Customizable rendering steps
Visitor Query DSL (IQuery, IQueryVisitor) Multi-row variable set queries
Builder CatalogTemplate, ProducerTargetVariableMapper.Builder Fluent construction
Repository / Cache ServiceCatalogCacheUtils, CatalogUtil Domain-aware data access
