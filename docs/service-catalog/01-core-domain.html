Checkout Flow

checkoutCart()
  → validate items (UserCriteria, domain)
  → create sc_request record
  → for each CartItem:
      → create sc_req_item record
      → copy variable answers
      → ProducerValve (if record producer type)
  → Request.recalculatePrice()
  → Request.recalculateRecurringCost()
  → return Request

Domain Conflict Detection

CartRecord enforces single-domain carts. If items from different domains are added, findConflictingDomain() returns the conflict for the UI to display.

Key Methods

Method Purpose

Request(sysId, preGr, curGr) Constructor with update tracking
recalculatePrice() Sum all item prices × quantities
recalculateRecurringCost() Aggregate recurring costs by frequency into sc_recurring_rollup
cancelRelatedRecords() Cancel items and approvals
cloneForUser(userId) Deep-copy request for another user

Price Recalculation Logic

On full recalculation:
  sum(item.price × quantity) for items where includePrice() == true
  → handles multi-currency via GlideElementCurrency
  → writes total to sc_request.price

On item update:
  subtract previous item contribution
  add current item contribution

On item delete:
  subtract deleted item contribution

Recurring Cost Rollup

  • Aggregates recurring prices by frequency string ("monthly", "quarterly", etc.)
  • Creates/updates sc_recurring_rollup records (one per frequency)
  • On full recalculation: deletes existing rollups, regenerates from scratch

Cancellation Rules

  • Never cancels items in closed_complete state
  • Respects glide.sc.cancel_closed_incomplete_ritm property
  • Cascades to sysapproval_approver records (bulk cancel via updateMultiple())

OrderGuideRule

Conditional logic controlling which items appear in the guide. Rules evaluate against variable answers from previous steps and filter IncludedItem objects.

IncludedItem

Represents one catalog item bundled into an order guide, with: - fCatalogItem — the item - fRule — the associated OrderGuideRule - fVariableAssignments — pre-filled variable values - fCascadeVariableMap — variable propagation map

Variable Mapping (cascade_variable_map)

When a user fills a variable on one guide item, the cascade map propagates that value to matching variables on other guide items automatically.

Sequencing Integration

When fSequenced=true, after checkout the OrderGuideSequencing class creates a PAD (Process/Activity Designer) process: 1. Creates a workflow trigger filtered by order_guide + approval status 2. Creates one activity (lane) per included item 3. Items execute as parallel PAD activities
