Overview

Catalog Builder is presented as a guided way to create catalog items and catalog item templates. Instead of asking a builder to manually configure many classic Service Catalog records, the experience organizes the work into wizard steps, templates, questions, settings, access rules, and fulfillment choices.

Core mental model: Catalog Builder is a guided front door. The real implementation is still backed by ServiceNow records such as catalog items, wizard steps, questions, templates, producer sets, and fulfillment step configurations.

Architecture: how Catalog Builder is built

The demo explicitly breaks Catalog Builder into five major building blocks:

Building blockRole in the demo
Catalog BuilderA guided experience for creating catalog items and catalog item templates without exposing every classic catalog configuration record directly.
UI Builder / UIBThe shell and dashboard experience: cards, lists, modals, event handlers, routes, and client scripts.
Catalog WizardThe ordered guided flow. The demo shows a Create catalog item wizard with Details, Location, Questions, Settings, Access, Fulfillment, and Review.
Wizard SectionsStep-level sections that group questions and content inside the wizard.
Questions / VariablesRequester-facing fields created through the question modal. They become the catalog item questions users answer during request submission.
Catalog Item TemplatesReusable templates that predefine values and constraints, making item creation faster and more governed.
OverridesTemplate controls that decide what the user can edit, what must be specified, and which multi-select options are allowed.
Composite Record ProducerBackend record producer layer that collects wizard answers and produces one or more target records.
Producer SetA grouped producer configuration, used in the demo for multi-row/many-to-many category mapping.

Create catalog item flow

The Create catalog item flow guides the builder through a series of steps. Each step captures a specific part of the item definition.

StepWhat it capturesWhy it matters
DetailsName, short description, image, and description.Controls how the item appears to requesters.
LocationCatalog and category placement.Controls browse/search discoverability.
QuestionsVariables/questions users answer.Collects request-specific data.
SettingsSubmit label and cart/request behavior.Shapes the portal checkout experience.
AccessWho can see or use the item.Controls availability and governance.
FulfillmentTask or approval steps.Turns an item request into operational work.
Review and submitSummary and final validation.Prevents incomplete or wrong configurations.

Catalog item templates and overrides

Templates let admins provide safe starting points for common catalog item patterns. A template can predefine values, control scope, restrict availability, and decide what the builder is allowed or required to change.

Important distinction: A template is not only a starter form. It is also a governance mechanism because it can lock fields, require fields, and restrict multi-select options.

Fulfillment configuration

The fulfillment step selects a fulfillment process and adds fulfillment steps. The backend uses reusable fulfillment step configurations such as Task, Custom approval, and Manager approval. Each option points to a fulfillment subflow.
